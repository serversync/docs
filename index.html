Docker

INTRODUÇÃO AO DOCKER COMPOSE

O Docker Compose é uma ferramenta para definir e executar aplicações Docker multi-container. Com o Compose, você usa um arquivo YAML para configurar os serviços da sua aplicação e, com um único comando, cria e inicia todos os serviços.
Cloudflare Tunnel

O Cloudflare Tunnel oferece uma conexão segura entre seus serviços e a Cloudflare:

version: '3.8'

                            services:
                            cloudflared:
                                image: cloudflare/cloudflared:latest
                                restart: unless-stopped
                                command: tunnel run
                                environment:
                                - TUNNEL_TOKEN=seu-token
                                volumes:
                                - ./cloudflared:/etc/cloudflared

Instruções de Uso:

  1. Substitua 'seu-token' pelo token gerado no painel da Cloudflare
  2. O diretório ./cloudflared será criado automaticamente
  3. Execute com: docker-compose up -d
Vaultwarden (Gerenciador de Senhas)

Vaultwarden é uma implementação alternativa do Bitwarden em Rust:

version: '3.8'

services:
  vaultwarden:
    image: vaultwarden/server:latest
    container_name: vaultwarden
    restart: unless-stopped
    environment:
      SMTP_HOST: "smtp.hostinger.com"
      SMTP_PORT: 587
      SMTP_SECURITY: "starttls"
      SMTP_USERNAME: "exemple@exemple.xyz"
      SMTP_PASSWORD: 'ezemple'
      SMTP_FROM: "exemple@exemple.xyz"
      SMTP_FROM_NAME: "Vaultwarden"
      SIGNUPS_VERIFY: "false"
      SIGNUPS_ALLOWED: "true"
      INVITATIONS_ALLOWED: "true"
      DOMAIN: "https://vw.domain.tld"
    volumes:
      - /home/ubuntu/vault:/data/
    ports:
      - 80:8082

Configurações Importantes:

  • SMTP: Configurações para envio de emails
  • SIGNUPS_VERIFY: Verificação de email no registro
  • DOMAIN: URL de acesso ao serviço
  • Volumes: Local de armazenamento dos dados
Dicas e Boas Práticas

Segurança:

  • Nunca exponha senhas diretamente no docker-compose.yml
  • Use arquivos .env para variáveis sensíveis
  • Limite o acesso às portas expostas
  • Mantenha as imagens atualizadas

Performance:

  • Use volumes nomeados em vez de bind mounts
  • Configure limites de recursos quando necessário
  • Otimize as camadas das imagens

Comandos Úteis:

# Iniciar serviços
docker-compose up -d

# Parar serviços
docker-compose down

# Visualizar logs
docker-compose logs -f

# Atualizar imagens
docker-compose pull

Documentação da GameServer:

SOBRE O SERVER STATUS CHECKER

O Server Status Checker é uma ferramenta poderosa para monitoramento de servidores de jogos. Ele exibe informações detalhadas sobre o status de servidores, como IP, porta, status (online/offline), jogadores online, ping e outros detalhes importantes. Além disso, o sistema permite a integração com o Discord, enviando atualizações automáticas de status para um canal específico via webhooks.

FUNCIONABILIDADES

  • 👉 Monitoramento de Servidores de Jogos: Verifique servidores de diferentes jogos, como Minecraft, CS:GO, Valheim e muitos outros.
  • 👉 Informações Detalhadas: Obtenha dados como número de jogadores, ping, status (online/offline) e jogadores conectados
  • 👉 Integração com Discord: Receba atualizações automáticas de status diretamente em seu canal Discord via webhooks.
  • 👉 Interface Web para Gerenciamento: Permita que os usuários adicionem servidores manualmente e visualizem seu status em tempo real.

TECNOLOGIAS USADAS

  • 👉 Node.js: Plataforma de desenvolvimento para o backend.
  • 👉 Express: Framework para construir o servidor e gerenciar rotas.
  • 👉 GameDig: Biblioteca para consultar o status de servidores de jogos como Minecraft, CS:GO, Valheim, entre outros.
  • 👉 Discord Webhooks: Para enviar atualizações de status para canais no Discord.
  • 👉 HTML, CSS, JavaScript: Para a construção da interface web.
  • 👉 File System (fs): Para armazenar configurações de servidores e cache localmente.

Arquivos Principais

server.js

Controle o servidor web e gerencie as rotas. As principais funções incluem:

  • 👉 Carregar e salvar servidores: Armazena os servidores em um arquivo servers.json.
  • 👉 Gerenciar cache de status: O cache de status dos servidores é atualizado periodicamente.
  • 👉 Rota /add-server: Permite adicionar novos servidores ao sistema e atualizar seu status.
  • 👉 Rota /remaining-time: Retorna o tempo restante para a próxima atualização de status.
  • 👉 Notificações para Discord: Envia atualizações para o Discord via webhook quando o status de um servidor é alterado.

dig.js

Responsável pela consulta do status dos servidores:

  • 👉 getCachedServerStatuses: Recebe uma lista de servidores e consulta o status (online, ping, jogadores, etc.).
  • 👉 addServer: Adiciona novos servidores à lista de monitoramento.

discordWebhook.js

Gerencia a integração com o Discord:

  • 👉 sendToDiscord: Envia uma mensagem formatada com as informações de status do servidor para o Discord.

servers.json

Arquivo de configuração:

  • 👉 Armazena a lista de servidores monitorados
  • 👉 Contém informações como IP, porta, tipo de jogo e webhook do Discord

cache.json

Armazenamento temporário:

  • 👉 Armazena o status mais recente dos servidores
  • 👉 Inclui número de jogadores, ping, status (online/offline) e outros detalhes

public/index.html

Interface web do sistema:

  • 👉 Exibição de Status: Mostra informações como IP, porta, tipo de jogo, número de jogadores e status
  • 👉 Adicionar Servidores: Permite adicionar servidores com IP, porta, tipo de jogo e webhook do Discord
  • 👉 Contagem Regressiva: Exibe o tempo restante até a próxima atualização de status

public/users.json

Gerenciamento de usuários:

  • 👉 Armazena as credenciais dos usuários para autenticação
  • 👉 Contém um array de usuários com nome de usuário e senha

Perguntas Frequentes

Docker

Como resolver problemas de permissão no Docker?

Para resolver problemas de permissão, você pode:

# Adicionar seu usuário ao grupo docker
sudo usermod -aG docker $USER

# Reiniciar o serviço
sudo systemctl restart docker

# Recarregar grupos
newgrp docker

Como limpar recursos não utilizados do Docker?

Para limpar imagens, containers e volumes não utilizados:

# Remover todos os containers parados
docker container prune

# Remover imagens não utilizadas
docker image prune -a

# Remover volumes não utilizados
docker volume prune

# Limpar tudo de uma vez
docker system prune -a

Como fazer backup dos dados do container?

Para fazer backup dos volumes do Docker:

# Criar backup de um volume
docker run --rm -v volume_name:/source -v $(pwd):/backup alpine tar -czf /backup/volume_backup.tar.gz -C /source .
GameServer

Como adicionar um novo servidor para monitoramento?

Para adicionar um novo servidor:

  1. Acesse a interface web
  2. Clique em "Adicionar Servidor"
  3. Preencha os campos:
    • IP do servidor
    • Porta
    • Tipo de jogo
    • Webhook do Discord (opcional)
  4. Clique em "Salvar"

Como configurar as notificações do Discord?

Para configurar as notificações:

  1. No Discord, vá para as configurações do canal
  2. Integrações > Webhooks > Criar Webhook
  3. Copie a URL do webhook
  4. Cole a URL na configuração do servidor no GameServer

O que fazer se o servidor não responde?

Se um servidor não responde, verifique:

  • Se o IP e porta estão corretos
  • Se o servidor está online
  • Se há firewall bloqueando o acesso
  • Se o tipo de jogo está configurado corretamente
Scripts e Sistema

Como atualizar o sistema?

Para atualizar o sistema, execute:

npm run update

Como fazer backup da configuração?

Para fazer backup dos arquivos de configuração:

# Backup dos arquivos de configuração
cp servers.json servers.json.backup
cp cache.json cache.json.backup
cp public/users.json public/users.json.backup

Como resolver problemas de permissão nos scripts?

Se encontrar problemas de permissão ao executar scripts:

# Dar permissão de execução
chmod +x script.sh

# Se necessário, executar como sudo
sudo ./script.sh
Rede e Conectividade

Como verificar se as portas estão abertas?

Para verificar se uma porta está acessível:

# Verificar porta local
netstat -tuln | grep PORTA

# Verificar porta remota
nc -zv IP PORTA

# Verificar com telnet
telnet IP PORTA

Como configurar o firewall?

Para configurar o firewall UFW:

# Permitir porta específica
sudo ufw allow PORTA

# Permitir porta para IP específico
sudo ufw allow from IP to any port PORTA

# Ativar firewall
sudo ufw enable
